## Description

Foodio backend API built with NestJS, TypeORM, PostgreSQL, JWT auth, and role-based access control.

## Workspace Scripts

Run from `backend`:

```bash
npm run start
npm run start:dev
npm run start:prod
npm run lint
npm run type-check
npm run validate
npm run test
npm run test:e2e
npm run test:cov
```

Or from monorepo root:

```bash
npm run start:dev -w backend
npm run validate -w backend
npm run test -w backend
```

## Validation & Security Notes

- Global `ValidationPipe` is enabled with `whitelist`, `forbidNonWhitelisted`, and `transform`.
- Core endpoints now use DTO validation for categories, menu items, and orders.
- Order placement validates item availability and quantity constraints.
- Order status updates enforce forward transitions:
  - `Pending -> Preparing -> Ready -> Completed`
- RBAC is enforced via JWT guard + roles guard for admin-only operations.

## Environment

Use `.env` in `backend` (see root README for full example). Minimum required variables:

```env
DB_HOST=localhost
DB_PORT=5432
DB_USERNAME=postgres
DB_PASSWORD=postgres
DB_NAME=foodio
JWT_SECRET=supersecret
PORT=3000
```
